// Week by Week
Full Curriculum — 6 Weeks
  • Why MLOps exists: the ML engineering debt problem
  • MLflow: tracking server setup, logging runs, artefacts
  • Model registry: versioning, stage transitions (Staging → Production)
  • MLflow Model Serving: serve registered models as REST endpoints
  • Feature stores: what they are and why they matter at scale
  • Feast: define feature views, materialise, retrieve online features
  • Avoiding training-serving skew — the #1 production bug
  • From notebooks to reproducible pipelines — why scripts aren't enough
  • Metaflow: DAG-style ML workflows, @step, @retry, @batch
  • Prefect: flows, tasks, deployments, schedules, error handling
  • Data versioning with DVC — track datasets like Git tracks code
  • Pipeline testing: unit testing transforms, integration testing pipelines
  • Parameterised pipeline runs — grid search at pipeline level
  • Build: end-to-end training pipeline from raw data to registered model
  • Docker fundamentals: images, layers, volumes, networking
  • Dockerising an ML model: base image selection, dependency management
  • FastAPI: build a production-grade prediction API with validation
  • Async serving: concurrent requests, thread pools, async def endpoints
  • NVIDIA Triton Inference Server: batching, model ensembles, gRPC
  • Docker Compose: multi-service ML stacks (API + model + monitoring)
  • Benchmarking latency and throughput: locust load testing
  • AWS SageMaker: training jobs, managed endpoints, auto-scaling
  • SageMaker Pipelines: cloud-native ML workflow orchestration
  • GCP Vertex AI: custom training, prediction endpoints, batch jobs
  • Cloud Run: serverless container deployment for lighter workloads
  • Kubernetes basics: pods, deployments, services, HPA
  • A/B testing model versions: traffic splitting on cloud endpoints
  • Cost optimisation: spot instances, autoscaling, request batching
  • CI/CD fundamentals applied to ML: what changes versus regular software
  • GitHub Actions: workflow triggers, matrix strategies, secrets
  • ML-specific CI: data validation, model quality gates, eval metric thresholds
  • Automated retraining trigger: push → test → train → evaluate → gate → deploy
  • Model cards: documenting intended use, limitations, fairness evaluation
  • Blue-green and canary deployments for ML models
  • Security: model signing, artefact provenance, SBOM for ML
  • What can go wrong in production: data drift, concept drift, upstream changes
  • Evidently: generate HTML drift reports, data quality checks
  • Prometheus: custom ML metrics, scraping prediction endpoints
  • Grafana: dashboards for model performance, drift alerts, business KPIs
  • Continuous training (CT): drift alert → retrain trigger → pipeline → deploy
  • ML system design patterns: two-tower, near-real-time, batch serving
  • Capstone: build and deploy a complete ML system with full CI/CD/CT loop
